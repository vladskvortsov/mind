Повідомлення від Тім Ліда (Дмитро Гриценко):  
Дата: 24 вересня 2024 року

Огляд проекту:  
Нам потрібно створити онлайн-каталог рецептів, де користувачі можуть переглядати рецепти та ділитися власними ідеями. Додаток повинен мати надійний бекенд для управління рецептами та користувачами, статичний фронтенд та API для запитів щодо пошуку рецептів і додавання нових рецептів.

Задача:  
Задеплоїти бекенд, API, Redis для кешування, RDS для бази даних, а також налаштувати статичний фронтенд через S3 та CloudFront згідно з наданою архітектурою. Завдання – забезпечити роботу сервісів у рамках виділеного бюджету.

База даних:  
Для управління інформацією про рецепти та користувачів ми будемо використовувати PostgreSQL. Redis допоможе в кешуванні результатів пошуку та сесій користувачів для підвищення продуктивності.

Докеризація:  
Усі сервіси мають бути докеризовані для зручного розгортання та масштабування.

CI/CD процес:  
Необхідно впровадити CI/CD пайплайни для автоматизації деплою бекенду та фронтенду.

Важливі нотатки:  
Фронтенд має бути статичним і розміщуватися в S3, кешування здійснюється через CloudFront. Ви повинні задеплоїти всю архітектуру на базі EC2 або ECS, як зазначено на діаграмі. Додаток повинен відповідати вимогам бюджету та забезпечувати масштабованість.
